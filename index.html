This is the Stencila development server for the /web module.

Javascript tests

The test suite can be run in the browser here. (You can also run then in Node.js using make test, npm test, or node tests)

Test pages

Snippets for individual node types are wrapped and served up with the correct JS and CSS. These are good for development of individual node types - without the distraction of a larger document containing many nodes type (several of which might also be calling out for attention). e.g.

  • /tests/document/nodes/codeblock
  • /tests/document/nodes/math

Static examples

Static HTML files that mimic the structure of Stencila components are available:

  • Document: /examples/document-design/
  • Stencil: /examples/a/
  • Sheet: /examples/b/

Other

This server bundles and compiles Javascript and CSS from local source files for convenience during development.

All other requests are proxied to an upstream server, by default https://localhost:7373 (a locally running Stencila session) but overridable at the command line e.g. node server.js https://stenci.la. This allows you to test the JS and SCSS using live Stencila components. The Javascript and CSS still gets generated on the fly from the local source but everything else gets passed on to the backend session. Make sure you add the trailing slash to avoid a redirect from the upstream e.g.

  • /demo/stencils/simple-r/
  • /demo/sheets/simple-r/
  • /demo/sheets/iris/
